## Issue Summary

GitHub issue #344, `refactor: introduce seed newtypes for domain separation (MaterialSeed, PlacementSeed, etc.)`, requires deterministic generation seed domains to use Rust newtypes rather than interchangeable bare `u64` values.

The issue body names the initial affected areas:

- `src/materials.rs` — `MaterialCatalog` keyed by `MaterialSeed` instead of `u64`.
- `src/observation.rs` — `ObsKey` uses `MaterialSeed`.
- `src/world_generation.rs` — `WorldProfile` sub-seeds get proper types.
- `src/solar_system.rs` — system/planet seed types.
- All derivation call sites.

The owner clarification in the issue comments strengthens the requirement: every value that semantically represents a deterministic generation seed must have an enforced domain type. This includes derived sub-seeds, stored generation keys, registries, caches, observation keys, generated identifiers, and helper boundaries where seed-domain values cross. Raw `u64` is only acceptable at explicit config/save/debug serialization boundaries, seed utility internals, and local hashing/mixing code where bits are actually combined.
